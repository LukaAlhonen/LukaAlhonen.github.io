Temperature monitoring dashboard

The point of this project was initially to do something with graphql subscriptions and thus I decided to create a distributed temperature monitoring system.
The data is collected by the mqtt publishers, written in rust with the rumqttc library. An eclipse mqtt broker then forwards the temperature readings to the graphql server which also runs an mqtt subscriber client.
The server then serves both the graphql subscriptions and writes the data to the influxdb3 time-series database.
The dashboard client is built using react, apollo client and typescript

Simulated soil-moisture monitoring system

A distributed, automated irrigation system I built since I wanted to create an IoT application using Rust.

The system uses mqtt for communication between worker nodes, which are written in rust with the rumqttc library. The publisher clients "read" from a simulated sensor, which currently is just a function that return a random value. However, with a few more lines of code, the simulated sensor could theoretically be susbstituted for a real one. The subscribers log all the received messages into a time-series database (influxdb). There is currently no frontend for the system, however I am planning on creating one when I have the time.

Additionally I built a ci/cd pipeline that builds and tests the code + builds and publishes the publisher and subscriber images and deploys using k3s

The github repository can be found here
